__init__(channel, time, deployment='history', frac_time_digits=0, sig_figs=6, data_updates_only=False, forward_time_search=False, exclude_given_time=False, enums_as_strings=False, unix_timestamps_ms=False, adjust_time_to_server_offset=False, **kwargs)[source]

Construct a query to the myquery interval service.

Parameters:

  • channel (str) – A list of PV Names to query

  • time (datetime) – The start time of the query

  • deployment (Optional[str]) – The mya deployment to query

  • frac_time_digits (int) – How many digits should be displayed for fractional seconds

  • sig_figs (int) – How many significant figures should be reported in the PV values

  • data_updates_only (bool) – Should the response include updates that only include value changes (not disconnects)

  • forward_time_search (bool) – Look forward in time from the given time for the next event (if True)

  • exclude_given_time (bool) – Don’t include the given time in the search space for the next event (if True)

  • enums_as_strings (bool) – Should enum PV values be returned as their named strings instead of ints

  • unix_timestamps_ms (bool) – Should timestamps be returned as millis since unix epoch

  • adjust_time_to_server_offset (bool) – Should the timestamp be localized to the myquery server

  • integrate – Should the values be integrated (ony supported for float PVs)

  • kwargs – Any extra parameters to be supplied to the interval web end point. Will produce a warning if used to avoid accidental use.
